## Problem Statement

### What Problem Does This Paper Solve?
LLM serving requires efficient attention kernels, but:
- **Variable sequence lengths**: Requests have different lengths
- **Dynamic batching**: Batch composition changes over time
- **Memory efficiency**: KV cache management critical
- **Hardware utilization**: Need to saturate GPU compute

### Why Is This Problem Important?
- LLM serving is expensive (compute costs dominate)
- Attention is the bottleneck in transformer inference
- Existing kernels optimized for fixed shapes
- Production needs handle dynamic workloads

### What Were the Limitations of Prior Work?
- **FlashAttention**: Optimized for training (fixed batch)
- **PagedAttention**: Memory management but kernel overhead
- **Custom kernels**: Often limited to specific use cases
- **No unified library** for production LLM serving

## Technical Contributions

### Core Innovation
FlashInfer provides **production-quality attention kernels** with:
1. Optimized variable-length attention
2. Efficient batching strategies
3. Memory-efficient KV cache handling
4. State-of-the-art GPU utilization

### Key Techniques & Methods

#### 1. Variable-Length Attention Kernels
- **Ragged tensor support**: Handle different sequence lengths
- **Load balancing**: Efficient work distribution across warps
- **Coalesced memory access**: Optimize global memory patterns
- **Shared memory optimization**: Cache Q, K, V tiles

#### 2. Batching Strategies
- **Request batching**: Group compatible requests
- **Chunked attention**: Process long sequences in chunks
- **Priority scheduling**: Support QoS requirements
- **Preemption**: Handle variable priority requests

#### 3. KV Cache Management
- **Page-based allocation**: Efficient memory reuse
- **Prefix caching**: Share common prefixes
- **Compression**: Quantization for cache storage
- **Eviction policies**: LRU and custom policies

#### 4. Kernel Optimizations
- **CUDA warp specialization**: Different warps for different tasks
- **Tensor Core utilization**: Mixed-precision computation
- **Asynchronous operations**: Hide memory latency
- **Kernel fusion**: Combine operations where possible

### System Architecture
- **Kernel library**: CUDA kernels for attention variants
- **C++ runtime**: Efficient scheduling and memory management
- **Python bindings**: Easy integration with serving frameworks
- **Integration**: Works with vLLM, SGLang, etc.

## Experimental Results

### Evaluation Setup
- **Models**: Llama, GPT, various sizes
- **Workloads**: Production traces, synthetic benchmarks
- **Hardware**: NVIDIA A100, H100
- **Baselines**: FlashAttention, PagedAttention, custom kernels

### Key Results
- **Throughput**: Up to 2x improvement over FlashAttention for serving
- **Latency**: Reduced P99 latency under load
- **Memory efficiency**: Better cache utilization
- **Flexibility**: Supports various attention variants

Performance gains from:
