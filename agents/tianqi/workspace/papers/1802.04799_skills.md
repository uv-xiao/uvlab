## Problem Statement

### What Problem Does This Paper Solve?
Deploying deep learning models on diverse hardware (CPUs, GPUs, mobile accelerators) requires:
- Hardware-specific optimizations
- Manual tuning for each target
- Different backends for different platforms
- Performance portability across hardware

### Why Is This Problem Important?
- ML models deployed on diverse devices (cloud servers to mobile phones)
- Hardware specialization increasing (TPUs, NPUs, custom accelerators)
- Manual optimization doesn't scale
- Need unified optimization stack

### What Were the Limitations of Prior Work?
- **CUDA/cuDNN**: NVIDIA only
- **Halide**: Requires manual schedules
- **TensorFlow/XLA**: Limited to specific backends
- **No unified solution** for diverse hardware

## Technical Contributions

### Core Innovation
TVM provides a **unified compiler stack** that:
1. Represents ML computations as computational graphs
2. Generates optimized code for diverse hardware
3. Learns optimization strategies automatically
4. Supports end-to-end model compilation

### Key Techniques & Methods

#### 1. Computational Graph Optimization
- **Operator fusion**: Combine multiple ops into single kernel
- **Constant folding**: Pre-compute constant expressions
- **Dead code elimination**: Remove unused computations
- **Layout transformation**: Optimize tensor layouts

#### 2. Tensor Expression Language
- **Schedule primitives**: Loop transformations (split, fuse, reorder)
- **Hardware intrinsics**: Map to SIMD, vector units
- **Memory scopes**: Manage cache/shared memory
- **Thread binding**: Map to GPU threads/blocks

#### 3. Automated Optimization
- **ML-based cost models**: Predict operator performance
- **Search algorithms**: Evolutionary and learned search
- **AutoTVM**: Automatic tuning for specific workloads
- **Transfer learning**: Reuse tuning across similar shapes

#### 4. Runtime System
- **Graph runtime**: Execute optimized graphs
- **Packed functions**: Cross-language function calls
- **Memory management**: Efficient buffer allocation

### System Architecture
- **Front-end**: Import from TF, PyTorch, MXNet, ONNX
- **Relay**: High-level IR for graph optimization
- **TE (Tensor Expression)**: Low-level operator description
- **TIR**: Target-independent representation
- **Codegen**: Generate hardware-specific code

## Experimental Results

### Evaluation Setup
- **Models**: ResNet, MobileNet, LSTM, etc.
- **Hardware**: x86, ARM, NVIDIA GPU, Mali GPU, FPGA
- **Baselines**: vendor libraries (cuDNN, MKL-DNN), hand-optimized

### Key Results
- **GPU**: Competitive with cuDNN for many workloads
- **ARM**: 2x faster than ARM Compute Library
- **FPGA**: Enabled efficient deployment
- **Mobile**: Efficient execution on resource-constrained devices

### Analysis of Results
Success factors:
1. Hardware-agnostic high-level optimizations
2. Learned cost models for target-specific tuning
3. Flexible operator fusion
4. Active open-source community
