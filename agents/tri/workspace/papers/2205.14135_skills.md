## Problem Statement

### What Problem Does This Paper Solve?
Standard attention implementation is inefficient because:
- **Memory bottleneck**: HBM bandwidth limits performance
- **Memory usage**: O(N²) memory for attention matrix
- **Redundant computation**: Materializing full attention matrix unnecessary
- **No hardware awareness**: Doesn't optimize for memory hierarchy

### Why Is This Problem Important?
- Attention dominates transformer runtime and memory
- Long sequences increasingly important (GPT-4, Claude)
- Memory limits model size and sequence length
- Efficiency critical for training cost and inference latency

### What Were the Limitations of Prior Work?
- **Standard implementation**: Materializes full N×N attention matrix
- **Sparse attention**: Approximate, doesn't match quality
- **Linear attention**: Approximate, different complexity trade-offs
- **No exact algorithm** that is memory-efficient and fast

## Technical Contributions

### Core Innovation
FlashAttention provides **IO-aware exact attention** that:
1. Computes attention without materializing full N×N matrix
2. Uses tiling to fit in SRAM (cache)
3. Reduces HBM reads/writes from O(N²) to O(N)
4. Achieves up to 7.6x speedup on A100

### Key Techniques & Methods

#### 1. Tiling Strategy
- **Block decomposition**: Split Q, K, V into blocks
- **SRAM residency**: Keep blocks in fast on-chip memory
- **Output accumulation**: Accumulate partial attention outputs
- **Softmax rescaling**: Online softmax computation

#### 2. Online Softmax
- **Incremental computation**: Compute softmax without full matrix
- **Rescaling factors**: Track max and sum for correct normalization
- **Numerical stability**: Careful handling of exponentials

#### 3. Recomputation for Backward Pass
- **Don't store attention matrix**: Recompute during backward
- **Store softmax statistics**: Only save normalization factors
- **Memory vs compute trade-off**: Extra compute saves memory

#### 4. IO-Aware Implementation
- **Memory hierarchy awareness**: Optimize for HBM, SRAM, registers
- **Kernel fusion**: Combine operations to reduce memory traffic
- **Warp scheduling**: Optimize thread block configuration

### System Architecture
- **Forward kernel**: Tiled attention computation
- **Backward kernel**: Gradient computation with recomputation
- **CUDA implementation**: Optimized for NVIDIA GPUs
- **Drop-in replacement**: API-compatible with standard attention

## Experimental Results

### Evaluation Setup
- **Models**: GPT-2, GPT-NeoX, various sizes
- **Sequence lengths**: 512 to 64K tokens
- **Hardware**: A100 GPUs
- **Baselines**: Standard PyTorch, memory-efficient attention

### Key Results
- **Speedup**: 2-4x faster than standard attention
- **Memory**: O(N) instead of O(N²)
- **Longer sequences**: Enable training with 16K+ tokens
- **Exact**: No approximation, same output quality

### Analysis of Results
Key insights:
1. Memory bandwidth, not compute, is the bottleneck
2. Tiling + online softmax enables memory efficiency
3. Exact attention can be faster than approximate methods
4. Hardware-aware algorithm design crucial
