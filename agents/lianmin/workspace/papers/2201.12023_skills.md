## Problem Statement

### What Problem Does This Paper Solve?
Training large neural networks requires parallelism across multiple devices, but choosing the right parallelization strategy is complex:
- **Data parallelism**: Replicate model, split data
- **Model parallelism (intra-op)**: Split operators across devices
- **Pipeline parallelism (inter-op)**: Split layers across devices
- **Combinations**: Complex search space of hybrid strategies

### Why Is This Problem Important?
- Model sizes growing exponentially (GPT-3 has 175B parameters)
- Manual parallelization requires expertise and trial-and-error
- Wrong strategy → poor utilization, slow training
- Different models need different strategies

### What Were the Limitations of Prior Work?
- **Mesh-TensorFlow**: Requires manual specification
- **GSPMD**: Automated but limited to certain parallelism types
- **PipeDream**: Focused only on pipeline parallelism
- **No system** automatically searched across all parallelism dimensions

## Technical Contributions

### Core Innovation
Alpa automatically generates **optimal parallelization strategies** by:
1. Viewing parallelization as integer linear programming
2. Defining parallelization as sharding specifications
3. Searching across inter-op and intra-op dimensions
4. Generating execution plans automatically

### Key Techniques & Methods

#### 1. Parallelism View as Sharding
- **Tensor sharding**: How each tensor is split across devices
- **Operator sharding**: Derives from input tensor shardings
- **Device mesh**: 2D mesh of devices (can be virtualized)

#### 2. Inter-Operator Parallelism (Pipeline)
- **Layer assignment**: Which layers on which devices
- **Stage construction**: Grouping layers into pipeline stages
- **Scheduling**: Forward/backward pass coordination

#### 3. Intra-Operator Parallelism
- **Data parallelism**: Shard batch dimension
- **Tensor parallelism**: Shard parameter dimensions
- **Hybrid**: Multiple dimensions sharded simultaneously

#### 4. Auto-Parallelization Algorithm
- **Cost model**: Estimates execution time for different strategies
- **Search algorithm**: Dynamic programming over computation graph
- **Optimization**: ILP formulation for layer assignment

### System Architecture
- **Front-end**: JAX/XLA integration
- **Compiler**: Generates parallel execution plan
- **Runtime**: Distributed execution on device cluster

## Experimental Results

### Evaluation Setup
- **Models**: GPT-3, GShard, Wide-ResNet
- **Hardware**: Multi-node GPU clusters
- **Baselines**: Megatron-LM, DeepSpeed, manual strategies

### Key Results
- **GPT-3 175B**: Competitive with hand-tuned Megatron-LM
- **Automatic discovery**: Found optimal strategies without manual tuning
- **Scalability**: Efficient scaling to hundreds of GPUs

The system successfully:
