## Problem Statement

### What Problem Does This Paper Solve?
Programming LLMs efficiently is challenging due to:
1. Complex control flows (branching, loops) in multi-turn conversations
2. Need for structured generation (JSON, code, etc.)
3. Batch processing with dynamic sequences
4. KV cache management across requests

### Why Is This Problem Important?
- LLM applications need structured outputs
- Existing frameworks require manual batching and cache management
- Performance is critical for production deployments
- Developer productivity matters for complex LLM programs

### What Were the Limitations of Prior Work?
- **vLLM**: Good for simple serving, limited programming model
- **Guidance/Outlines**: Structured generation but limited control flow
- **Raw APIs**: Too low-level, manual optimization required

## Technical Contributions

### Core Innovation
SGLang provides an **embedded domain-specific language** for programming LLMs with:
- **Structured generation primitives**: Regex, JSON, CFG constraints
- **Control flow**: Native Python control flow (if/else, loops)
- **Automatic batching**: Runtime automatically batches operations
- **KV cache sharing**: RadixAttention for automatic prefix caching

### Key Techniques & Methods

#### 1. Structured Generation DSL
- **Regex constraints**: Generate text matching patterns
- **JSON schema**: Valid JSON output generation
- **Context-free grammars**: Complex structure validation
- **Efficient decoding**: Constrained token masking

#### 2. RadixAttention
- **Prefix caching**: Automatically cache common prefixes
- **KV cache management**: Efficient reuse across requests
- **Radix tree structure**: Organize caches by shared prefixes
- **Automatic eviction**: LRU policy for memory management

#### 3. Runtime Batching
- **Continuous batching**: Add new requests dynamically
- **Speculative execution**: Generate multiple tokens speculatively
- **Automatic fusion**: Combine compatible operations

### System Architecture
- **Frontend**: Python DSL embedded in host language
- **Runtime**: Efficient scheduling and execution engine
- **Backend**: Integration with optimized kernels (FlashAttention, etc.)

## Experimental Results

### Evaluation Setup
- **Benchmarks**: Various LLM programming tasks
- **Baselines**: vLLM, Guidance, raw APIs
- **Metrics**: Throughput, latency, code complexity

### Key Results
- **Throughput**: Up to 5x improvement over vLLM
- **Latency**: Reduced for structured generation
- **Programming ease**: Significantly less code required

### Analysis of Results
The gains come from:
1. Automatic batching reducing GPU idle time
2. KV cache reuse via RadixAttention
3. Optimized structured generation kernels
